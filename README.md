#客服文本分类测试项目

##一、原始代码Code Review问题（按严重程度排序）

###问题1：原始代码中将LLM接口密钥直接写死在代码内，上传代码仓库会造成凭证泄露，产生额外扣费风险。
影响：任何人获取代码即可调用本人大模型接口，造成资金损失。

###问题2：原始Prompt仅简单下达分类指令，未提供类目释义、区分逻辑，大模型易混淆相近客服场景。
影响：30条测试样本中错分样本较多，基准准确率仅93.33%。

###问题3：调用LLM接口未处理网络超时、返回格式异常等场景。
影响：批量跑测试样本时一旦报错，程序直接终止，无法完整完成评估。

##二、整体改进思路
###1. 针对密钥泄露问题：将代码内真实API密钥替换为占位文字，完成脱敏处理；
###2. 针对Prompt缺陷重构提示词：补充类目释义、细化类别边界、规范输出格式、增加兜底判定规则；
###3. 使用30条标注测试样本运行评估脚本，对比优化前后分类准确率，验证优化效果。

##Prompt改动理由
###1. 关联'task1_categories.md'补充全部类目详细释义，明确每一类客服话术的区分判定标准；
###2. 细化各类别边界区分逻辑，厘清退款、投诉等易混淆场景的判定优先级，减少分类错判；
###3. 强制限定输出格式，要求仅返回类目名称，剔除多余解释文字，便于评估脚本自动解析结果；
###4. 增加兜底规则，文本无法判定归属时统一返回「未知」，防止程序解析报错。

##三、准确率对比（30条标注测试样本）
优化前基准准确率：93.33%
Prompt优化后准确率：96.67%
提升幅度：3.34%

##四、AI工具使用情况
###1. Prompt优化：使用AI辅助重构分类提示词，完善类目判定规则；
###2. 评估脚本：AI辅助编写'eval.py'批量评测代码，实现自动统计准确率。
